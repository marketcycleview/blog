# SEO 블로그 프로젝트 가이드라인

## 프로젝트 개요

애드센스 수익화를 목표로 한 SEO 최적화 정보 + 리뷰 통합 블로그

### 핵심 컨셉
- 정보성 콘텐츠 + 자동 수집 리뷰 통합
- 한국어/영어 다국어 지원
- 프로그램적 SEO + 트렌딩 콘텐츠 혼합

---

## 기술 스택

```
Framework: Next.js 15 (App Router, SSG/ISR)
Styling: Tailwind CSS
Content: MDX (Contentlayer 또는 next-mdx-remote)
Deploy: Vercel
Analytics: Google Analytics 4 + Search Console
```

## 콘텐츠 생성 가이드라인

### ⚠️ AI스럽지 않게 쓰기 (최우선 규칙)

> **핵심**: 사람이 쓴 것처럼 자연스럽게. AI 티 나면 실패.

#### 🚫 절대 하지 말 것 (AI 특징)

```
❌ 금지 패턴:
- "~에 대해 알아보겠습니다" "~을 살펴보겠습니다" (도입부 상투어)
- "결론적으로", "따라서", "그러므로" (과도한 연결어)
- "다양한", "효과적인", "중요한", "필수적인" (형용사 남용)
- "~라고 할 수 있습니다" (애매한 표현)
- "이 글에서는 ~를 다룹니다" (메타 설명)
- 모든 문장을 "~입니다"로 끝내기
- 완벽하게 정렬된 글머리 기호 (3개씩 딱딱 맞추기)
- 감정 없이 정보만 나열
```

#### ✅ 이렇게 써야 함

**1. 바로 본론으로**
```
❌ "민생지원금에 대해 알아보겠습니다. 민생지원금이란..."
✅ "민생지원금, 25만원 받을 수 있습니다. 신청 안 하면 그냥 날아가요."
```

**2. 구어체 섞기**
```
❌ "신청 방법은 다음과 같습니다."
✅ "신청 방법 진짜 간단해요." / "신청은 이렇게 하면 됩니다."
```

**3. 개인 의견/경험 넣기**
```
❌ "이 제품은 쿠션감이 우수합니다."
✅ "솔직히 처음 신었을 때 '이게 뭐지?' 싶었는데, 5km 뛰고 나니까 진가를 알겠더라고요."
```

**4. 문장 길이 불규칙하게**
```
❌ "이 제품의 장점은 A입니다. 또한 B라는 장점이 있습니다. 그리고 C도 있습니다."
✅ "장점? 일단 가볍습니다. 280g. 그리고 쿠션이 푹신한데 반발력도 있어요. 근데 제일 좋은 건 가격이에요."
```

**5. 의문형/감탄형 활용**
```
❌ "많은 분들이 궁금해하시는 부분입니다."
✅ "근데 이거 진짜 궁금하잖아요?" / "이게 핵심이에요!"
```

**6. 불완전한 문장도 OK**
```
❌ "가격은 159,000원입니다."
✅ "가격? 159,000원." / "가격은 15만 9천원. 생각보다 괜찮죠?"
```

#### 톤앤매너 가이드

| 카테고리 | 톤 | 예시 |
|---------|-----|------|
| 지원금 | 친절한 공무원 | "놓치면 아까우니까 꼭 신청하세요" |
| 리뷰 | 솔직한 친구 | "솔직히 말하면..." "개인적으로는..." |
| 트렌딩 | 뉴스+수다 | "이거 대박인데요?" "논란이 되고 있어요" |

#### 자연스러운 연결어

```
AI가 쓰는 연결어 → 사람이 쓰는 연결어
"따라서" → "그래서" / "그러니까"
"그러나" → "근데" / "다만"
"또한" → "그리고" / "또"
"예를 들어" → "예를 들면" / "가령"
"결론적으로" → "정리하면" / "결국"
"특히" → "특히나" / "뭐니뭐니 해도"
```

#### 문장 끝 변화주기

```
~입니다 / ~이에요 / ~거든요 / ~잖아요 / ~인 거죠
~해요 / ~합니다 / ~하더라고요 / ~한다고요
```

**3문장 연속으로 같은 어미 금지!**

#### 실제 예시 비교

**❌ AI스러운 글:**
```
뉴발란스 1080v13에 대해 알아보겠습니다.
이 제품은 러닝화로, 다양한 장점을 가지고 있습니다.
첫째, 쿠션감이 우수합니다.
둘째, 안정성이 뛰어납니다.
셋째, 디자인이 세련되었습니다.
따라서 러닝을 즐기시는 분들께 추천드립니다.
```

**✅ 자연스러운 글:**
```
뉴발란스 1080v13, 6개월 신고 다녔습니다.

결론부터 말하면? 족저근막염 있는 분들한테 강추예요.
쿠션이 푹신한데 발이 안 빠지는 느낌? 이게 생각보다 중요하거든요.

근데 단점도 있어요. 일단 무거워요. 310g이니까
"가벼운 러닝화 찾아요"하시는 분들은 패스하세요.

가격은 159,000원. 비싸긴 한데... 발 편한 거 생각하면
그냥 지르는 게 맞는 것 같아요.
```

### MDX 콘텐츠 작성 규칙

#### 표(Table) 작성 가이드

**필수 규칙**:
- 표는 반드시 마크다운 표 문법 사용
- 모든 열에 헤더 행 포함
- 셀 내용은 간결하게 (긴 내용은 줄바꿈 X)
- 모바일 대응을 위해 열 수는 4개 이하 권장

**올바른 표 형식**:
```markdown
| 항목 | 내용 | 비고 |
|------|------|------|
| 지원금액 | 25만원 | 1인당 |
| 신청기간 | 2026.01.15 ~ 02.28 | 선착순 아님 |
| 지급일 | 2026.03.15 | 예정 |
```

**잘못된 표 형식 (사용 금지)**:
```markdown
항목: 지원금액
내용: 25만원
(이런 식으로 쓰지 말 것)
```

**스펙 비교표 예시**:
```markdown
| 항목 | 제품A | 제품B |
|------|-------|-------|
| 가격 | 159,000원 | 189,000원 |
| 무게 | 280g | 310g |
| 쿠션 | Fresh Foam X | React |
| 추천 | 가성비 | 성능 |
```

#### 이미지 삽입 가이드

**이미지 파일 위치**:
- 내부 이미지: `/public/images/[category]/[파일명]`
- 외부 이미지: 직접 URL 사용 가능

**이미지 삽입 문법**:
```markdown
![이미지 설명 (alt 텍스트)](/images/subsidy/minsaeng-2026.png)

또는 외부 이미지:
![제품 사진](https://example.com/product.jpg)
```

**이미지 필수 규칙**:
- alt 텍스트 필수 (SEO + 접근성)
- 파일명은 영문 소문자, 하이픈 사용 (예: `newbalance-1080v13-side.jpg`)
- 용량 최적화 (500KB 이하 권장)
- 주요 이미지는 가로 1200px 이상

**권장 이미지 수**:
- 지원금 글: 2-4장 (신청 화면 캡처)
- 제품 리뷰: 5-10장 (다양한 각도)
- 트렌딩: 1-3장 (핵심 이미지)

**이미지 수집 방법** (에이전트용):
```
웹 검색으로 이미지 찾을 때:
1. "[키워드] site:정부24" - 공식 이미지
2. "[제품명] 실제 사진" - 실사용 이미지
3. 이미지 URL을 MDX에 직접 삽입
4. 저작권 주의 (공식 사이트, 보도자료 우선)
```

#### 광고 삽입 가이드

**광고 위치 (자동 삽입)**:
- 글 상단: 제목 아래, 본문 시작 전
- 글 하단: 본문 끝, 관련 글 위

**본문 중간 광고 (선택)**:
MDX에서 직접 광고 컴포넌트 삽입 가능:
```markdown
## 섹션 1 내용...

<AdPlaceholder slot="middle" />

## 섹션 2 내용...
```

**광고 삽입 규칙**:
- 최소 300단어마다 1개 (너무 많으면 사용자 경험 저하)
- H2 제목 바로 위에 배치 권장
- 표나 이미지 바로 옆에는 배치 X

**효과적인 광고 위치**:
```
1. 제목 바로 아래 (조회수 높음)
2. 목차 아래
3. 본문 중간 (스크롤 50% 지점)
4. 결론 위
5. 관련 글 섹션 위
```

### 콘텐츠 유형별 가이드

---

#### 1. 지원금/정책 정보 (subsidy)

**목적**: 정부/지자체 지원금, 복지 정책, 세금 관련 정보 제공

**글 구조**:
```
1. 핵심 요약 (지원금액, 신청기간 - 표로 정리)
2. 지원 대상/자격 요건
3. 신청 방법 (단계별 스크린샷 포함)
4. 필요 서류
5. 지급 일정
6. 주의사항/꿀팁
7. FAQ
8. 실제 수혜자 반응 (리뷰 섹션)
```

**필수 포함 요소**:
- 공식 출처 링크 (정부24, 복지로 등)
- 신청 기간 명확히 (시작일 ~ 마감일)
- 연도 명시 필수 (2026년)
- 지역별 차이 있으면 명시

**SEO 키워드 패턴**:
```
- [지원금명] 신청방법
- [지원금명] 자격요건
- [지원금명] 신청기간
- [연도] [지원금명] 총정리
- [지원금명] 대상 확인
```

**예시 제목**:
```
- 2026년 민생지원금 신청방법 총정리 (자격요건, 신청기간)
- 청년내일저축계좌 신청자격 확인하기 (2026년 최신)
- 경기도 청년기본소득 신청방법 5단계 가이드
```

**주의사항**:
- 정보 변경 시 즉시 업데이트 필요
- 잘못된 정보 제공 시 신뢰도 하락
- 마감된 정책은 "마감됨" 표시 또는 삭제

**수익성**: ★★★★★ (높음 - 검색량 많고 광고 단가 높음)

#### 2. 제품 리뷰 (review)

**목적**: 제품 구매 결정에 도움되는 종합 리뷰 제공

**글 구조**:
```
1. 한줄 결론 (이 제품 추천하는 사람/추천 안하는 사람)
2. 제품 기본 정보 (스펙 표)
3. 장점 (3-5개)
4. 단점 (2-3개)
5. 실사용 후기 요약
6. 가격 비교 (쿠팡, 네이버 등)
7. 대안 제품 추천
8. 수집된 리뷰 (긍정/부정 분류)
9. 총평 및 별점
```

**필수 포함 요소**:
- 제품 스펙 표
- 가격 정보 (업데이트 날짜 명시)
- 실제 사용자 리뷰 인용
- 비교 제품 언급
- 구매 링크

**SEO 키워드 패턴**:
```
- [제품명] 후기
- [제품명] 장단점
- [제품명] 실사용 리뷰
- [제품명] vs [경쟁제품]
- [제품명] 구매 전 알아야 할 것
- [카테고리] 추천 [연도]
```

**예시 제목**:
```
- 뉴발란스 1080v13 실착 후기 (6개월 사용, 장단점 총정리)
- 에어팟 프로 2 솔직 리뷰 - 구매 전 꼭 알아야 할 5가지
- 다이슨 V15 vs V12 비교 - 어떤 게 나을까?
```

**서브 카테고리**:
```
/review/
├── /shoes/          # 신발 (러닝화, 운동화, 구두)
├── /electronics/    # 가전 (청소기, TV, 냉장고)
├── /mobile/         # 모바일 (스마트폰, 태블릿, 이어폰)
├── /beauty/         # 뷰티 (화장품, 스킨케어)
├── /kitchen/        # 주방 (에어프라이어, 커피머신)
├── /furniture/      # 가구 (의자, 책상, 매트리스)
└── /etc/            # 기타
```

**주의사항**:
- 협찬/광고 표시 필수 (해당 시)
- 가격 정보는 변동될 수 있음 명시
- 과장된 표현 자제

**수익성**: ★★★★☆ (높음 - 구매 의도 높은 검색)

---

#### 3. 트렌딩/이슈 (trending)

**목적**: 실시간 화제, 뉴스, 이슈에 대한 빠른 정보 제공

**글 구조**:
```
1. 핵심 요약 (무슨 일? 3줄 요약)
2. 상세 내용 (육하원칙)
3. 배경 설명
4. 전문가/관계자 반응
5. 네티즌 반응 (긍정/부정)
6. 향후 전망
7. 관련 이슈 링크
```

**필수 포함 요소**:
- 발생 일시 명확히
- 출처 명시 (어디서 보도됐는지)
- 실시간 업데이트 표시
- SNS/커뮤니티 반응

**SEO 키워드 패턴**:
```
- [이슈] 정리
- [이슈] 원인
- [이슈] 반응
- [인물] 논란 정리
- [사건] 총정리
```

**예시 제목**:
```
- [속보] 미국 한파 피해 현황 총정리 (2026년 1월)
- OO 논란 정리 - 무슨 일이 있었나? (타임라인)
- 주식시장 급락 원인 3가지 분석
```

**주의사항**:
- 속도가 생명 (빠른 발행)
- 확인되지 않은 정보 "~로 알려졌다" 표현
- 명예훼손 주의
- 자극적 제목 자제

**수익성**: ★★★☆☆ (중간 - 트래픽 높지만 일시적)

   └── SEO 키워드 확장

2. 웹 리서치 (모든 사이트에서 수집)
   ├── "[키워드]" 검색
   ├── "[키워드] 후기/리뷰" 검색
   ├── "[키워드] 장단점" 검색
   └── 공식 출처 확인

3. 정보 & 리뷰 수집
   ├── 핵심 정보 추출
   ├── 리뷰/반응 수집 (긍정/부정 분류)
   ├── 모든 출처 URL 기록
   └── 신뢰도 높은 정보 우선
