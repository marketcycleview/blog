# SEO 블로그 프로젝트 가이드라인

## 프로젝트 개요

애드센스 수익화를 목표로 한 SEO 최적화 정보 블로그

### 🎯 사이트 테마: "내 돈 챙기기"

> **핵심 방향**: 복지/금융/세금 정보에 집중. 테마를 벗어난 콘텐츠는 작성하지 않음.

```
✅ 작성 가능한 주제:
├── 복지/지원금: 정부 지원금, 복지 정책, 보조금
├── 금융: 대출, 저축, 투자(정보성), 금융상품 비교, 주식(정보성)
├── 세금: 연말정산, 종합소득세, 양도세, 세금 신고
└── 계산기/도구: 연봉 계산기, 대출 이자 계산기, 세금 계산기

❌ 작성하지 않는 주제:
├── 제품 리뷰 (신발, 가전 등)
├── 여행 정보
├── 건강/의료 정보
├── 트렌딩/연예 이슈
├── 맛집/지역 정보
└── 기술/IT (금융과 무관한)
```

### 핵심 컨셉
- 복지/금융/세금 정보 전문 블로그
- 한국어/영어 다국어 지원
- 프로그램적 SEO (계산기, 도구 페이지)

### ⚠️ 글쓰기 톤 가이드 (최우선 규칙)

> **핵심**: "친절한 전문가"처럼 쓰기. AI스럽지도, 너무 캐주얼하지도 않게.

#### 🎯 목표: 친절한 전문가 톤

```
❌ AI스러운 글: 딱딱함, 신뢰↓, Google AI 감지 위험
❌ 너무 구어체: 전문성↓, 개인 블로그 느낌
✅ 친절한 전문가: 정확한 정보 + 자연스러운 설명
```

#### 🚫 절대 금지 (AI 특징)

```
- "~에 대해 알아보겠습니다" (도입부 상투어)
- "결론적으로", "따라서", "그러므로" 남발
- "다양한", "효과적인", "중요한" 형용사 반복
- "~라고 할 수 있습니다" (애매한 표현)
- 3문장 연속 같은 어미 ("~입니다. ~입니다. ~입니다.")
- 완벽하게 정렬된 글머리 (첫째/둘째/셋째)

#### 💡 자연스럽게 쓰는 팁

**1. 바로 본론으로**

```

"그러나" → "다만" / "근데"
"또한" → "그리고"
"결론적으로" → "정리하면"

**3. 어미 변화주기**
```
~입니다 / ~이에요 / ~거든요 / ~인데요 / ~세요
(3문장 연속 같은 어미 금지)
```

**4. 중간중간 독자에게 말 걸기**
```

### 🔥 지원금/정책 글 필수: eligibility 메타데이터

> **중요**: subsidy 카테고리 글 작성 시 반드시 `eligibility`와 `benefit` 필드를 포함해야 합니다.
> 이 데이터는 `/ko/tools/welfare-finder` 복지 정책 찾기 도구에서 필터링에 사용됩니다.

#### 지원금/정책 MDX 템플릿 (전체)

```yaml
---
title: "2026년 [정책명] 총정리 (자격요건, 지원금액, 신청방법)"
description: "메타 설명 155자 이내"
date: "2026-01-28"
category: "subsidy"
tags: ["정책명", "관련태그", "2026"]
locale: "ko"
keywords: ["정책명 신청방법", "정책명 자격요건"]

# === 필터링용 메타데이터 (필수!) ===
eligibility:
  age:                          # 나이 조건 (선택)
    min: 19                     # 최소 나이
    max: 34                     # 최대 나이
    note: "군필자 최대 39세"     # 예외사항 메모
  income:                       # 소득 조건 (선택)
    type: "median"              # median(중위소득%) | fixed(고정금액) | none(무관)
    percent: 60                 # 중위소득 60% 이하
    maxAmount: 7000             # 고정금액일 경우 (만원 단위)
    note: "청년가구 기준"
  asset:                        # 재산 조건 (선택)
    max: 12200                  # 만원 단위
  targetGroups:                 # 대상 그룹 (필수! 최소 1개)
    - "youth"                   # 아래 코드표 참조
    - "renter"
  housing:                      # 주거 형태 (선택)
    - "rent"
    - "jeonse"
  householdType:                # 가구 유형 (선택)
    - "single"
    - "couple"
  region: null                  # 지역 제한 (null = 전국)
  disabilityRequired: false     # 장애인 필수 여부

benefit:
  amount: 200000                # 지원금액 (원 단위)
  duration: 24                  # 지원기간 (개월)
  type: "monthly"               # monthly | onetime | yearly | loan | other
  note: "월 최대 20만원"
---
```

#### targetGroups 코드표 (대상 그룹)

| 코드 | 대상 | 사용 예시 |
|------|------|----------|
| `youth` | 청년 (만 19~34세) | 청년월세지원, 청년도약계좌 |
| `student` | 대학생 | 학자금대출 |
| `jobseeker` | 구직자/실업자 | 실업급여, 구직촉진수당 |
| `worker` | 직장인 | 육아휴직급여, 근로장려금 |
| `selfemployed` | 자영업자 | 소상공인바우처 |
| `senior` | 노인 (만 65세+) | 기초연금, 국민연금 |
| `disabled` | 장애인 | 장애인연금 |
| `singleparent` | 한부모 | 한부모가족지원금 |
| `multicultural` | 다문화가정 | 다문화가정 지원 |
| `pregnant` | 임산부 | 첫만남이용권 |
| `infant` | 영유아 (0~6세) | 부모급여, 아이돌봄 |
| `child` | 아동 (7~17세) | 아동수당 |
| `veteran` | 국가유공자 | 보훈 관련 |
| `farmer` | 농업인 | 농어촌기본소득 |
| `lowIncome` | 저소득층 | 생계급여, 의료급여 |
| `renter` | 임차인 | 주거급여, 버팀목대출 |

#### housing 코드표 (주거 형태)

| 코드 | 의미 |
|------|------|
| `own` | 자가 |
| `jeonse` | 전세 |
| `rent` | 월세 |
| `homeless` | 무주택 |
| `public` | 공공임대 |

#### householdType 코드표 (가구 유형)

| 코드 | 의미 |
|------|------|
| `single` | 1인 가구 |
| `couple` | 신혼부부 |

#### 지원금 글 작성 예시

```yaml
# 예시 1: 청년월세지원
eligibility:
  age:
    min: 19
    max: 34
  income:
    type: "median"
    percent: 60
  targetGroups:
    - "youth"
    - "renter"
  housing:
    - "rent"
benefit:
  amount: 200000
  duration: 24
  type: "monthly"

# 예시 2: 기초연금
eligibility:
  age:
    min: 65
  income:
    type: "fixed"
    maxAmount: 247
  targetGroups:
    - "senior"
benefit:
  amount: 349360
  type: "monthly"

# 예시 3: 장애인연금 (장애 필수)
eligibility:
  targetGroups:
    - "disabled"
  disabilityRequired: true
benefit:
  amount: 439700
  type: "monthly"

# 예시 4: 전기차 보조금 (대상 제한 없음)
eligibility:
  income:
    type: "none"
  targetGroups: []
benefit:
  amount: 6800000
  type: "onetime"
```

#### ⚠️ 주의사항

1. **targetGroups는 필수**: 빈 배열 `[]`이라도 반드시 포함
2. **금액은 원 단위**: 20만원 → `200000`
3. **기간은 개월 단위**: 2년 → `24`
4. **region이 null이면 전국**: 특정 지역만이면 `"seoul"`, `"gyeonggi"` 등 사용
5. **정보성 글도 메타데이터 포함**: targetGroups를 빈 배열로 설정

### MDX 콘텐츠 작성 규칙

#### 표(Table) 작성 가이드

**필수 규칙**:
- 표는 반드시 마크다운 표 문법 사용
- 모든 열에 헤더 행 포함
- 셀 내용은 간결하게 (긴 내용은 줄바꿈 X)
- 모바일 대응을 위해 열 수는 4개 이하 권장

**올바른 표 형식**:
```markdown
| 항목 | 내용 | 비고 |
|------|------|------|
| 지원금액 | 25만원 | 1인당 |
| 신청기간 | 2026.01.15 ~ 02.28 | 선착순 아님 |
| 지급일 | 2026.03.15 | 예정 |
```

**잘못된 표 형식 (사용 금지)**:
```markdown
항목: 지원금액
내용: 25만원
(이런 식으로 쓰지 말 것)
```

**스펙 비교표 예시**:
```markdown
| 항목 | 제품A | 제품B |
|------|-------|-------|
| 가격 | 159,000원 | 189,000원 |
| 무게 | 280g | 310g |
| 쿠션 | Fresh Foam X | React |
| 추천 | 가성비 | 성능 |
```

#### 이미지 가이드

##### 1. 동적 OG 이미지 (썸네일) - 자동 생성

모든 글의 대표 이미지(썸네일)는 **@vercel/og**를 사용해 자동 생성됩니다.

```
┌─────────────────────────────────────┐
│  InfoTalker                         │
│                                     │
│  청년 월세 지원                      │
│  2026년 총정리                       │
│                                     │
│  💰 월 20만원 × 24개월               │
└─────────────────────────────────────┘
```

**자동 생성 방식**:
- 글 제목(title)과 설명(description)을 기반으로 생성
- 카테고리별 색상/아이콘 자동 적용
- Open Graph, Twitter Card에 자동 연결
- MDX에 별도 이미지 삽입 불필요

**카테고리별 디자인**:
| 카테고리 | 색상 | 아이콘 |
|---------|------|--------|

6. 주의사항/꿀팁
7. FAQ
8. 실제 수혜자 반응 (리뷰 섹션)

**필수 포함 요소**:

**SEO 키워드 패턴**:

```

**예시 제목**:

- 청년내일저축계좌 신청자격 확인하기 (2026년 최신)

**주의사항**:

- 잘못된 정보 제공 시 신뢰도 하락
- 마감된 정책은 "마감됨" 표시 또는 삭제

**수익성**: ★★★★★ (높음 - 검색량 많고 광고 단가 높음)

#### 3. How-to 가이드 (방법/튜토리얼)

**목적**: 특정 작업 수행 방법을 단계별로 안내

**글 구조**:
```
1. 이 가이드에서 배울 내용
2. 필요한 것 (준비물)
3. 단계별 가이드 (스크린샷/이미지 포함)
   - Step 1: ...
   - Step 2: ...
   - Step 3: ...
4. 주의사항/팁
5. 자주 하는 실수
6. FAQ
7. 관련 가이드 링크
```

**필수 포함 요소**:
- 단계별 번호 매기기
- 각 단계 스크린샷/이미지
- 예상 소요 시간
- 난이도 표시

**SEO 키워드 패턴**:
```
- [작업] 방법
- [작업] 하는 법
- [작업] 따라하기
- [작업] 초보 가이드
- [작업] 쉽게 하는 법
```

**예시 제목**:
```
- 주식 계좌 개설 방법 - 초보도 따라하는 가이드
- 홈택스 연말정산 간소화 서비스 사용법
- 청약통장 만드는 방법 (은행별 비교)
- 신용점수 올리는 방법 5가지 (실제 후기)
- 국민연금 예상 수령액 조회 방법
- 건강보험료 환급 신청 방법 (직장인/지역가입자)

**수익성**: ★★★★☆ (높음 - 검색량 안정적)

## 서브 키워드 & 콘텐츠 클러스터 전략

> 🎯 **핵심**: "총정리" 같은 누구나 쓰는 키워드만 쓰지 말고, **검색자가 실제로 겪는 문제 상황**을 키워드로 잡아라.

---

### 왜 서브 키워드인가?

```
❌ 일반 키워드만 쓰면:
- "아동수당 신청방법" → 경쟁 치열, 차별화 어려움
- 모든 블로그가 같은 키워드 타겟

✅ 서브 키워드 활용하면:
- "아동수당 반려됐을 때" → 실제 문제 겪는 사람이 검색
- "아동수당 신청 취소 방법" → 구체적 상황
- "아동수당 중복 신청 가능한가" → 궁금증 기반
- 경쟁 적고, 전환율 높음
```

---

### 서브 키워드 발굴 방법

**1. 문제 상황 기반**
```
메인 키워드: [지원금명]
서브 키워드:
├── [지원금명] 반려 사유
├── [지원금명] 거절됐을 때
├── [지원금명] 재신청 방법
├── [지원금명] 중복 수령 가능한가
├── [지원금명] 환수 조건
├── [지원금명] 신청 취소
└── [지원금명] 지급 안됨
```

**2. 검색자 질문 패턴**
```
- "~하면 어떻게 되나요?"
- "~할 수 있나요?"
- "~안 되는 이유"
- "~했는데 왜 안됨?"
- "~가 반려/거절되면?"
```

**3. 실제 예시 (아동수당)**
```
메인 글: 아동수당 신청방법 총정리

서브 글 (내부 링크로 연결):
├── 아동수당 반려 사유 5가지와 해결방법
├── 아동수당 중복 신청 시 어떻게 되나?
├── 아동수당 지급일 늦어지는 이유
├── 아동수당 소득 기준 초과 시 대안
├── 아동수당 해외 체류 시 수령 가능?
└── 아동수당 vs 양육수당 차이점
```

### 콘텐츠 클러스터 구조 (문어발 확장)

```
                    ┌─────────────────┐
                    │   메인 글 (허브)  │
                    │ "아동수당 총정리" │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
   ┌─────────┐         ┌─────────┐         ┌─────────┐
   │ 서브 글1 │         │ 서브 글2 │         │ 서브 글3 │
   │ 반려 사유│←───────→│ 지급일  │←───────→│중복 수령 │
   └─────────┘         └─────────┘         └─────────┘
        │                    │                    │
        ▼                    ▼                    ▼
   ┌─────────┐         ┌─────────┐         ┌─────────┐
   │재신청방법│         │ 소득기준 │         │ 해외체류│

```

### 제목 태그 (H1, H2, H3) 사용 규칙

#### H1 태그
```
- 페이지당 1개만 사용 (MDX에서 title이 H1)
- 주요 키워드 필수 포함
- 60자 이내 권장
```

#### H2 태그
```
- 주요 섹션 구분용
- 키워드 자연스럽게 포함 (2~3개 H2에)
- 글당 3~7개 권장
- 사용자가 훑어보기(스캔)할 때 글 구조 파악 가능해야 함
```

#### H3 태그
```
- H2 하위 세부 항목용
- 필요할 때만 사용 (무분별한 사용 X)
- H2 없이 H3 단독 사용 X
```

**올바른 구조 예시:**
```markdown
# 2026년 청년도약계좌 총정리 (H1 - title)

## 청년도약계좌란? (H2)

## 신청 자격 요건 (H2)
### 나이 조건 (H3)
### 소득 조건 (H3)

## 신청 방법 (H2)
### 온라인 신청 (H3)
### 오프라인 신청 (H3)

## 자주 묻는 질문 (H2)
```

**잘못된 구조 예시:**
```markdown
# 제목
### 바로 H3 사용 (X - H2 건너뜀)

## 섹션1
## 섹션2
## 섹션3
## 섹션4
## 섹션5
## 섹션6
## 섹션7
## 섹션8 (X - H2가 너무 많음)
```

### 키워드 최적화

#### 키워드 밀도
```
- 적정: 1~2%
- 과하면 (3%+): 키워드 스터핑으로 페널티
- 부족하면: 주제 관련성 인식 어려움
```

#### 키워드 배치 위치 (중요도 순)
```
1. 제목 (H1/title) - 필수
2. 메타 디스크립션 - 필수
3. 첫 100자(첫 문단) - 필수
4. H2 제목 2~3개 - 권장
5. 본문 중간중간 - 자연스럽게
6. 이미지 alt 태그 - 권장
7. URL slug - 권장
```

**예시 (키워드: "청년도약계좌 신청"):**
```markdown
---
title: "2026년 청년도약계좌 신청방법 총정리"  ← 키워드 포함

### 문단 및 가독성

#### 문단 길이
```
- 한 문단: 3~4문장 이하
- 긴 문단은 모바일에서 읽기 어려움
- 2~3줄마다 줄바꿈 권장
```

#### 가독성 향상 요소
```
✅ 표 (Table): 비교, 요약 정보에 필수
✅ 리스트 (불릿/숫자): 나열 정보에 사용
✅ 굵은 글씨 (**bold**): 핵심 키워드 강조
✅ 인용 (>): 중요 정보, 주의사항
✅ 코드블록 (```): 단계별 절차, 공식

❌ 긴 문단 (5문장+)
❌ 강조 없는 텍스트 덩어리
❌ 표/리스트 없이 텍스트만
```

### 이미지 SEO

```
- 글당 최소 2~3개 이미지 권장
- alt 태그 필수 (키워드 자연스럽게 포함)
- 파일명: 영문-키워드-포함.jpg (한글 X)
- 용량: 500KB 이하 (로딩 속도)
- 캡션 추가하면 더 좋음
```

**alt 태그 예시:**
```
✅ "청년도약계좌 신청 화면 캡처"
✅ "2026년 최저임금 인상률 그래프"

❌ "청년도약계좌청년도약계좌청년" (키워드 스터핑)
```

### 외부 링크 & LinkButton (URL 검증 필수)

> **핵심**: 외부 링크는 LinkButton 컴포넌트로 버튼 형식 사용. 클릭 유도 + 체류 시간 증가.

#### 기본 규칙
```
- 신뢰할 수 있는 출처만 (정부, 공식 기관)
- 글당 2~3개 권장
- LinkButton 컴포넌트 사용
- 메인 홈페이지 URL 금지 → 상세 페이지 URL 사용
```

#### LinkButton 사용법
```markdown
<LinkButton href="https://www.bokjiro.go.kr/specific-page" label="복지로에서 아동수당 신청하기" />
```

#### 버튼 라벨 예시
```
✅ 좋은 예:
- "복지로에서 아동수당 신청하기"
- "고용24에서 실업급여 조회하기"
- "홈택스에서 연말정산 간소화 서비스 이용하기"

❌ 나쁜 예:
- "여기 클릭"
- "바로가기"
- "링크"
```

#### ⚠️ URL 검증 필수 규칙 (매우 중요)

LinkButton에 URL을 추가하기 전에 **반드시** 해당 URL이 실제로 존재하는 페이지인지 확인해야 합니다.

```
검증 방법:
1. WebFetch 도구로 URL 접속 테스트
2. 404 에러, 리다이렉트, 페이지 없음 확인
3. 페이지 내용이 관련 주제와 일치하는지 확인

검증 체크리스트:
[ ] URL이 실제로 접속 가능한가?
[ ] 404 에러가 발생하지 않는가?
[ ] 페이지 내용이 버튼 라벨과 일치하는가?
[ ] 메인 홈페이지가 아닌 상세 페이지인가?
[ ] HTTPS를 사용하는가?
```

```
❌ 검증 없이 URL 추가 금지:
- 추측으로 URL 작성 (X)
- 예전에 봤던 URL 그대로 사용 (X)
- 비슷한 패턴으로 URL 생성 (X)

✅ 반드시 검증 후 추가:
- WebFetch로 페이지 존재 확인 (O)
- 실제 페이지 내용 확인 (O)
- 정확한 상세 페이지 URL 사용 (O)
```

#### 주요 사이트별 URL 패턴
```
복지로: https://www.bokjiro.go.kr/ssis-tbu/twataa/wlfareInfo/moveTWAT52011M.do?wlfareInfoId=WLF[숫자]
정부24: https://www.gov.kr/portal/rcvfvrSvc/dtlEx/[숫자]

홈택스: https://www.hometax.go.kr/websquare/websquare.html?w2xPath=[경로]
국민연금: https://www.nps.or.kr/jsppage/[경로]
```

#### 버튼 배치 규칙
```
- 각 섹션 끝에 관련 공식 링크 버튼 배치
- 신청 방법 섹션에는 반드시 "신청하러 가기" 버튼
- 본문 중간중간에 CTA 버튼 삽입 (2-3개 섹션마다)
- 모든 글 마지막에 "관련 링크" 섹션 필수

## UI 컴포넌트 가이드라인

### 검색창 (SearchBar)

**필수**: 모든 주요 페이지에 검색창을 포함해야 합니다.

```tsx
import { SearchBar } from "@/components/SearchBar";

// 사용법
<SearchBar locale={locale} />
```

**검색창이 필요한 페이지:**
- 메인페이지 (`/[locale]`) - 히어로 섹션
- 카테고리 페이지 (`/[locale]/[category]`) - 헤더 아래
- 검색 결과 페이지 (`/[locale]/search`) - 상단

**검색창이 불필요한 페이지:**
- 개별 글 페이지 (`/[locale]/[category]/[slug]`)
- 개인정보처리방침, 연락처 등 정적 페이지

### 글 목록 카드 (PostCard)

**필수**: 글 목록은 썸네일 이미지가 포함된 가로형 레이아웃을 사용합니다.

```
┌────────────┬──────────────────────────────┐
│            │ 💰 지원금 · 2026-01-27       │
│  🟦 썸네일  │ 글 제목                      │
│  (OG 이미지)│ 글 설명...                   │
└────────────┴──────────────────────────────┘
```

**적용 페이지:**
- 메인페이지 카테고리별 글 목록
- 카테고리 페이지 전체 글 목록

**썸네일 이미지:**
- `/api/og` 엔드포인트로 자동 생성
- 카테고리별 색상 (subsidy=파랑, finance=초록, tax=보라)
