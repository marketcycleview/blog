## 🚨 글 작성 시 필수 규칙 (최우선)

> **모든 글 작성 전 반드시 이 규칙을 따를 것**

### 1. 글 작성 전 준비

```
[ ] 이 CLAUDE.md 파일 전체 내용 숙지
[ ] 해당 카테고리 PLAN 파일에서 글 상태 확인 (중복 방지)
    - 복지: POLICY_WRITING_PLAN.md
    - 금융: FINANCE_WRITING_PLAN.md
    - 세금: TAX_WRITING_PLAN.md
    - 부동산: REAL_ESTATE_WRITING_PLAN.md
    - 커리어: CAREER_WRITING_PLAN.md
    - 법률: LEGAL_WRITING_PLAN.md
    - 창업: BUSINESS_WRITING_PLAN.md
[ ] 복지로/정부24에서 최신 정보 확인 (WebFetch 사용)
```

### 2. 링크 규칙 (매우 중요)

```
내부링크: 3~5개 (관련 정책 글로 연결)
외부링크(LinkButton): 1~3개 (공식 출처만)
총 링크: 최소 4개 ~ 최대 8개

외부링크 배치 위치 (1~3개 중 선택):
  - 핵심 요약 아래 또는 신청 방법 섹션
  - 관련 링크 섹션 (글 마지막)

❌ 금지: 버튼 5개 이상 과도하게 넣기
❌ 금지: 글 맨 끝에만 버튼 몰아넣기
❌ 금지: 검증 안 된 URL 사용
❌ 금지: 같은 URL 중복 사용 (파일 내 동일 URL 버튼 1개만)
```

### 3. URL 검증 필수

```
모든 LinkButton URL은 반드시 WebFetch로 검증 후 사용

검증 체크리스트:
[ ] URL이 실제로 접속 가능한가?
[ ] 404 에러가 발생하지 않는가?
[ ] 페이지 내용이 버튼 라벨과 일치하는가?
[ ] 메인 홈페이지가 아닌 상세 페이지인가?

❌ 추측으로 URL 작성 금지
❌ 예전에 봤던 URL 그대로 사용 금지
✅ 반드시 WebFetch로 확인 후 사용
```

### 4. 글 구조 체크리스트

```
[ ] 1500자 이상
[ ] 제목에 키워드 + 연도 포함
[ ] 첫 문단에 핵심 정보 (금액, 대상 등)
[ ] H2 3개 이상, H2 중 2개 이상에 키워드
[ ] 표(Table) 최소 2개 이상
[ ] FAQ 섹션 포함 (3~5개 질문)
[ ] 내부 링크 3개 이상 (관련 정책)
```

### 5. 글 작성 후 필수

```
[ ] POLICY_WRITING_PLAN.md 해당 항목 ✅ 완료로 변경
[ ] git add → commit → push (요청 시)
```

## 프로젝트 개요

애드센스 수익화를 목표로 한 SEO 최적화 정보 블로그

### 🎯 사이트 테마: "내 돈 챙기기"

> **핵심 방향**: 복지/금융/세금 정보에 집중. 테마를 벗어난 콘텐츠는 작성하지 않음.

```
✅ 작성 가능한 주제:
├── 복지/지원금: 정부 지원금, 복지 정책, 보조금
├── 금융: 대출, 저축, 투자(정보성), 금융상품 비교, 주식(정보성)
├── 세금: 연말정산, 종합소득세, 양도세, 세금 신고
├── 부동산: 전세/월세, 매매, 청약, 경매, 재개발
├── 취업/커리어: 연봉, 이직, 퇴사, 부업, 직장인 권리
├── 법률/소비자: 계약, 환불, 사기 대처, 소송, 내용증명
├── 창업/사업: 사업자등록, 온라인사업, 프랜차이즈, 운영
└── 계산기/도구: 연봉 계산기, 대출 이자 계산기, 세금 계산기

❌ 작성하지 않는 주제:
├── 제품 리뷰 (신발, 가전 등)
├── 여행 정보
├── 건강/의료 정보 (의료비 지원 제외)
├── 트렌딩/연예 이슈
├── 맛집/지역 정보
└── 기술/IT (금융/사업과 무관한)
```

### 핵심 컨셉
- 복지/금융/세금 정보 전문 블로그
- 한국어/영어 다국어 지원
- 프로그램적 SEO (계산기, 도구 페이지)

### 🔥 지원금/정책 글 필수: eligibility 메타데이터

> **중요**: subsidy 카테고리 글 작성 시 반드시 `eligibility`와 `benefit` 필드를 포함해야 합니다.
> 이 데이터는 `/ko/tools/welfare-finder` 복지 정책 찾기 도구에서 필터링에 사용됩니다.

#### 지원금/정책 MDX 템플릿 (전체)

```yaml
---
title: "2026년 [정책명] 총정리 (자격요건, 지원금액, 신청방법)"
description: "메타 설명 155자 이내"
date: "2026-01-28"
category: "subsidy"
tags: ["정책명", "관련태그", "2026"]
locale: "ko"
keywords: ["정책명 신청방법", "정책명 자격요건"]

# === 필터링용 메타데이터 (필수!) ===
eligibility:
  age:                          # 나이 조건 (선택)
    min: 19                     # 최소 나이
    max: 34                     # 최대 나이
    note: "군필자 최대 39세"     # 예외사항 메모
  income:                       # 소득 조건 (선택)
    type: "median"              # median(중위소득%) | fixed(고정금액) | none(무관)
    percent: 60                 # 중위소득 60% 이하
    maxAmount: 7000             # 고정금액일 경우 (만원 단위)
    note: "청년가구 기준"
  asset:                        # 재산 조건 (선택)
    max: 12200                  # 만원 단위
  targetGroups:                 # 대상 그룹 (필수! 최소 1개)
    - "youth"                   # 아래 코드표 참조
    - "renter"
  housing:                      # 주거 형태 (선택)
    - "rent"
    - "jeonse"
  householdType:                # 가구 유형 (선택)
    - "single"
    - "couple"
  region: null                  # 지역 제한 (null = 전국)
  disabilityRequired: false     # 장애인 필수 여부

benefit:
  amount: 200000                # 지원금액 (원 단위)
  duration: 24                  # 지원기간 (개월)
  type: "monthly"               # monthly | onetime | yearly | loan | other
  note: "월 최대 20만원"
---
```

#### targetGroups 코드표 (대상 그룹)

| 코드 | 대상 | 사용 예시 |
|------|------|----------|
| `youth` | 청년 (만 19~34세) | 청년월세지원, 청년도약계좌 |
| `student` | 대학생 | 학자금대출 |
| `jobseeker` | 구직자/실업자 | 실업급여, 구직촉진수당 |
| `worker` | 직장인 | 육아휴직급여, 근로장려금 |
| `selfemployed` | 자영업자 | 소상공인바우처 |
| `senior` | 노인 (만 65세+) | 기초연금, 국민연금 |
| `disabled` | 장애인 | 장애인연금 |
| `singleparent` | 한부모 | 한부모가족지원금 |
| `multicultural` | 다문화가정 | 다문화가정 지원 |
| `pregnant` | 임산부 | 첫만남이용권 |
| `infant` | 영유아 (0~6세) | 부모급여, 아이돌봄 |
| `child` | 아동 (7~17세) | 아동수당 |
| `veteran` | 국가유공자 | 보훈 관련 |
| `farmer` | 농업인 | 농어촌기본소득 |
| `lowIncome` | 저소득층 | 생계급여, 의료급여 |
| `renter` | 임차인 | 주거급여, 버팀목대출 |

#### housing 코드표 (주거 형태)

| 코드 | 의미 |
|------|------|
| `own` | 자가 |
| `jeonse` | 전세 |
| `rent` | 월세 |
| `homeless` | 무주택 |
| `public` | 공공임대 |

#### householdType 코드표 (가구 유형)

| 코드 | 의미 |
|------|------|
| `single` | 1인 가구 |
| `couple` | 신혼부부 |
| `family` | 일반 가족 |
| `multiChild` | 다자녀 (3자녀+) |
| `grandparent` | 조손가정 |
| `independent` | 독립 (부모 별거) |

#### 지원금 글 작성 예시

```yaml
# 예시 1: 청년월세지원
eligibility:
  age:
    min: 19
    max: 34
  income:
    type: "median"
    percent: 60
  targetGroups:
    - "youth"
    - "renter"
  housing:
    - "rent"
benefit:
  amount: 200000
  duration: 24
  type: "monthly"

# 예시 2: 기초연금
eligibility:
  age:
    min: 65
  income:
    type: "fixed"
    maxAmount: 247
  targetGroups:
    - "senior"
benefit:
  amount: 349360
  type: "monthly"

# 예시 3: 장애인연금 (장애 필수)
eligibility:
  targetGroups:
    - "disabled"
  disabilityRequired: true
benefit:
  amount: 439700
  type: "monthly"

# 예시 4: 전기차 보조금 (대상 제한 없음)
eligibility:
  income:
    type: "none"
  targetGroups: []
benefit:
  amount: 6800000
  type: "onetime"
```

#### ⚠️ 주의사항

1. **targetGroups는 필수**: 빈 배열 `[]`이라도 반드시 포함
2. **금액은 원 단위**: 20만원 → `200000`
3. **기간은 개월 단위**: 2년 → `24`
4. **region이 null이면 전국**: 특정 지역만이면 `"seoul"`, `"gyeonggi"` 등 사용
5. **정보성 글도 메타데이터 포함**: targetGroups를 빈 배열로 설정

### MDX 콘텐츠 작성 규칙

#### 표(Table) 작성 가이드

**필수 규칙**:
- 표는 반드시 마크다운 표 문법 사용
- 모든 열에 헤더 행 포함
- 셀 내용은 간결하게 (긴 내용은 줄바꿈 X)
- 모바일 대응을 위해 열 수는 4개 이하 권장

**올바른 표 형식**:
```markdown
| 항목 | 내용 | 비고 |
|------|------|------|
| 지원금액 | 25만원 | 1인당 |
| 신청기간 | 2026.01.15 ~ 02.28 | 선착순 아님 |
| 지급일 | 2026.03.15 | 예정 |
```

**잘못된 표 형식 (사용 금지)**:
```markdown
항목: 지원금액
내용: 25만원
(이런 식으로 쓰지 말 것)
```

**스펙 비교표 예시**:
```markdown
| 항목 | 제품A | 제품B |
|------|-------|-------|
| 가격 | 159,000원 | 189,000원 |
| 무게 | 280g | 310g |
| 쿠션 | Fresh Foam X | React |
| 추천 | 가성비 | 성능 |
```

#### 이미지 가이드

##### 1. 동적 OG 이미지 (썸네일) - 자동 생성

모든 글의 대표 이미지(썸네일)는 **@vercel/og**를 사용해 자동 생성됩니다.

```
┌─────────────────────────────────────┐
│  InfoTalker                         │
│                                     │
│  청년 월세 지원                      │
│  2026년 총정리                       │
│                                     │
│  💰 월 20만원 × 24개월               │
└─────────────────────────────────────┘
```

**자동 생성 방식**:
- 글 제목(title)과 설명(description)을 기반으로 생성
- 카테고리별 색상/아이콘 자동 적용
- Open Graph, Twitter Card에 자동 연결
- MDX에 별도 이미지 삽입 불필요

**카테고리별 디자인**:
| 카테고리 | 색상 | 아이콘 |
|---------|------|--------|
| subsidy | 파랑 (정부/공식) | 💰 |
| finance | 초록 (금융) | 🏦 |
| tax | 보라 (세금) | 📋 |
| real-estate | 주황 (부동산) | 🏠 |
| career | 남색 (커리어) | 💼 |
| legal | 빨강 (법률) | ⚖️ |
| business | 노랑 (창업) | 🚀 |

##### 2. 본문 이미지 (선택)

본문에 추가 이미지가 필요한 경우에만 삽입합니다.

**이미지 파일 위치**:
- 내부 이미지: `/public/images/[category]/[파일명]`
- 외부 이미지: 직접 URL 사용 가능

**이미지 삽입 문법**:
```markdown
![이미지 설명 (alt 텍스트)](/images/subsidy/minsaeng-2026.png)

또는 외부 이미지:
![제품 사진](https://example.com/product.jpg)
```

**권장 본문 이미지 수**:
- 지원금 글: 0-2장 (신청 화면 캡처 등)
- 금융 글: 0-2장 (금리 비교표, 신청 화면 등)
- 세금 글: 0-2장 (홈택스 화면 캡처 등)

### 제목 태그 (H1, H2, H3) 사용 규칙

#### H1 태그
```
- 페이지당 1개만 사용 (MDX에서 title이 H1)
- 주요 키워드 필수 포함
- 60자 이내 권장
```

#### H2 태그
```
- 주요 섹션 구분용
- 키워드 자연스럽게 포함 (2~3개 H2에)
- 글당 3~7개 권장
- 사용자가 훑어보기(스캔)할 때 글 구조 파악 가능해야 함
```

#### H3 태그
```
- H2 하위 세부 항목용
- 필요할 때만 사용 (무분별한 사용 X)
- H2 없이 H3 단독 사용 X
```

**올바른 구조 예시:**
```markdown
# 2026년 청년도약계좌 총정리 (H1 - title)

## 청년도약계좌란? (H2)

## 신청 자격 요건 (H2)
### 나이 조건 (H3)
### 소득 조건 (H3)

## 신청 방법 (H2)
### 온라인 신청 (H3)
### 오프라인 신청 (H3)

## 자주 묻는 질문 (H2)
```

**잘못된 구조 예시:**
```markdown
# 제목
### 바로 H3 사용 (X - H2 건너뜀)

## 섹션1
## 섹션2
## 섹션3
## 섹션4
## 섹션5
## 섹션6
## 섹션7
## 섹션8 (X - H2가 너무 많음)
```

### 키워드 최적화

#### 키워드 밀도
```
- 적정: 1~2%
- 과하면 (3%+): 키워드 스터핑으로 페널티
- 부족하면: 주제 관련성 인식 어려움
```

#### 키워드 배치 위치 (중요도 순)
```
1. 제목 (H1/title) - 필수
2. 메타 디스크립션 - 필수
3. 첫 100자(첫 문단) - 필수
4. H2 제목 2~3개 - 권장
5. 본문 중간중간 - 자연스럽게
6. 이미지 alt 태그 - 권장
7. URL slug - 권장
```

## 청년도약계좌 신청 자격  ← H2에 키워드

```

### 문단 및 가독성

#### 문단 길이
```
- 한 문단: 3~4문장 이하
- 긴 문단은 모바일에서 읽기 어려움
- 2~3줄마다 줄바꿈 권장
```

#### 가독성 향상 요소
```
✅ 표 (Table): 비교, 요약 정보에 필수
✅ 리스트 (불릿/숫자): 나열 정보에 사용
✅ 굵은 글씨 (**bold**): 핵심 키워드 강조
✅ 인용 (>): 중요 정보, 주의사항
✅ 코드블록 (```): 단계별 절차, 공식

❌ 긴 문단 (5문장+)
❌ 강조 없는 텍스트 덩어리
❌ 표/리스트 없이 텍스트만
```

## UI 컴포넌트 가이드라인

### 검색창 (SearchBar)

**필수**: 모든 주요 페이지에 검색창을 포함해야 합니다.

```tsx
import { SearchBar } from "@/components/SearchBar";

// 사용법
<SearchBar locale={locale} />
```

**검색창이 필요한 페이지:**
- 메인페이지 (`/[locale]`) - 히어로 섹션
- 카테고리 페이지 (`/[locale]/[category]`) - 헤더 아래
- 검색 결과 페이지 (`/[locale]/search`) - 상단

**검색창이 불필요한 페이지:**
- 개별 글 페이지 (`/[locale]/[category]/[slug]`)
- 개인정보처리방침, 연락처 등 정적 페이지

### 글 목록 카드 (PostCard)

**필수**: 글 목록은 썸네일 이미지가 포함된 가로형 레이아웃을 사용합니다.

```
┌────────────┬──────────────────────────────┐
│            │ 💰 지원금 · 2026-01-27       │
│  🟦 썸네일  │ 글 제목                      │
│  (OG 이미지)│ 글 설명...                   │
└────────────┴──────────────────────────────┘
```

**적용 페이지:**
- 메인페이지 카테고리별 글 목록
- 카테고리 페이지 전체 글 목록

**썸네일 이미지:**
- `/api/og` 엔드포인트로 자동 생성
- 카테고리별 색상 (subsidy=파랑, finance=초록, tax=보라)
