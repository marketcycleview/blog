## 글쓰기 톤 가이드

> **핵심**: "친절한 전문가"처럼 쓰기. AI스럽지도, 너무 캐주얼하지도 않게.

### 카테고리별 톤 비율

| 카테고리 | 문어체 | 친근함 | 컨셉 |
|---------|--------|--------|------|
| **복지/지원금** | 70% | 30% | 친절한 공무원 |
| **금융/대출** | 65% | 35% | 믿을 수 있는 상담사 |
| **세금/연말정산** | 70% | 30% | 쉽게 설명하는 세무사 |
| **비교 글** | 60% | 40% | 객관적인 분석가 |
| **How-to 가이드** | 55% | 45% | 친절한 선생님 |

### 절대 금지 (AI 특징)

```
- "~에 대해 알아보겠습니다" (도입부 상투어)
- "결론적으로", "따라서", "그러므로" 남발
- "다양한", "효과적인", "중요한" 형용사 반복
- "~라고 할 수 있습니다" (애매한 표현)
- 3문장 연속 같은 어미 ("~입니다. ~입니다. ~입니다.")
```

### 자연스럽게 쓰는 팁

**1. 바로 본론으로**
```
❌ "이번 글에서는 민생지원금에 대해 알아보겠습니다."
✅ "민생지원금, 25만원 받을 수 있습니다."
```

**2. 연결어 자연스럽게**
```
"따라서" → "그래서"
"그러나" → "다만" / "근데"
"또한" → "그리고"
"결론적으로" → "정리하면"
```

**3. 어미 변화주기**
```
~입니다 / ~이에요 / ~거든요 / ~인데요 / ~세요
(3문장 연속 같은 어미 금지)
```

**4. 글 시작 패턴 (돌려쓰기)**
```
패턴 1: "민생지원금, 1인당 25만원 지급됩니다." (핵심 먼저)
패턴 2: "민생지원금 신청, 어떻게 해야 할까요?" (질문형)
패턴 3: "2026년 민생지원금 신청이 2월부터 시작됩니다." (상황 제시)
패턴 4: "25만원. 신청만 하면 받을 수 있는 민생지원금입니다." (숫자로 시작)
```

⚠️ **같은 패턴 연속 사용 금지! 글마다 다르게 시작할 것.**

## 에이전트 실행 가이드

### 명령어 목록

| 명령어 예시 | 실행 동작 |
|------------|----------|
| "오늘 할 일 시작해볼까?" | 트렌드 분석 → 키워드 추천 → 글 작성 → 배포 |
| "오늘 트렌드 뭐야?" | 실시간 트렌딩 키워드 검색 및 추천 |
| "이 키워드로 글 써줘: [키워드]" | 해당 키워드로 글 작성 |
| "글 배포해줘" | git add, commit, push 실행 |

### 워크플로우: 글 작성

```
1. 키워드 분석 → 글 유형 판단
2. 웹 리서치 (공식 출처 확인)
3. 글 작성 (유형별 템플릿 적용)
4. 파일 저장: /content/{locale}/{category}/{slug}.mdx
5. 사용자 검토 요청
6. 승인 시 git 배포
```

## 서브 키워드 & 콘텐츠 클러스터 전략

> **핵심**: "총정리" 같은 누구나 쓰는 키워드만 쓰지 말고, **검색자가 실제로 겪는 문제 상황**을 키워드로 잡아라.

### 왜 서브 키워드인가?

```
❌ 일반 키워드: "아동수당 신청방법" → 경쟁 치열
✅ 서브 키워드: "아동수당 반려됐을 때" → 경쟁 적고 전환율 높음
```

### 서브 키워드 발굴 방법

**문제 상황 기반**:
```
[지원금명] 반려 사유
[지원금명] 거절됐을 때
[지원금명] 재신청 방법
[지원금명] 중복 수령 가능한가
```

**검색자 질문 패턴**:
```
"~하면 어떻게 되나요?"
"~할 수 있나요?"
"~안 되는 이유"
```

### 콘텐츠 클러스터 구조

```
        메인 글 (허브)
       "아동수당 총정리"
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 서브 글1   서브 글2   서브 글3
 반려 사유   지급일   중복 수령

→ 모든 글이 서로 내부 링크로 연결
→ 체류 시간↑, 페이지뷰↑, SEO↑
```

### 서브 키워드 확장 체크리스트

새 메인 글 작성 시, 아래 질문으로 서브 키워드 발굴:

```
[ ] 신청/가입이 거절/반려되면?
[ ] 중복으로 받을 수 있나?
[ ] 취소/해지하면 어떻게 되나?
[ ] 자격 안 되면 대안은?
[ ] vs 비슷한 다른 것과 차이점?
```

## 🧮 계산기 활용 콘텐츠 작성 가이드

> **핵심 원칙**: 계산기 링크를 자연스럽게 녹여서 체류시간 ↑, 도구 사용률 ↑

### 콘텐츠 유형 분류 (6가지)

| 유형 | 특징 | 예시 |
|------|------|------|
| **시뮬레이션형** | 특정 숫자 → 결과 | 연봉 3000 실수령액, 전세 3억 월세 전환 |

## 퇴직금 계산 3단계

### Step 1. 1일 평균임금 구하기
퇴직 전 3개월 총 임금 ÷ 그 기간의 총 일수
= (300만 × 3) ÷ 92일 = 약 97,826원

### Step 2. 30일 환산
97,826원 × 30일 = 약 293만원 (1년치 퇴직금)

### Step 3. 근속연수 곱하기
293만원 × 3년 = 약 879만원

### 금융/대출 정보 글 구조

```
1. 핵심 요약 (투자 주의 문구 포함)
2. 개념 설명
3. 장단점
4. 가입/신청 방법
5. 주의사항
6. 대안 상품 비교
7. FAQ
```

**필수 포함 요소**:
- 투자 위험 고지 문구
- 최신 금리/수익률 정보
- 공식 출처 링크

**SEO 키워드 패턴**: `[금융상품] 금리`, `[금융상품] 추천`, `[금융상품] 가입 방법`

### 금리/상품 비교표 글 구조

```
1. 이번 달 금리 요약 (TOP 5 표)
2. 은행별/상품별 상세 비교표
3. 가입 조건 정리
4. 추천 상품 (상황별)
5. 가입 방법/링크
```

**필수 포함 요소**:
- 업데이트 날짜 명시 (매우 중요!)
- 출처 명시 (은행 공식, 금융감독원)
- 우대금리 조건

**SEO 키워드 패턴**: `[연도][월] [상품] 금리`, `[상품] 금리 비교`

### 보험 정보 글 구조

```
1. 핵심 요약 (이 보험이 필요한 사람)
2. 보험 종류별 비교
3. 가입 시 체크포인트
4. 보험료 예시
5. 청구 방법
6. 주의사항/면책 조항
7. FAQ
```

**주의사항**:
- 특정 상품 추천 시 광고 표시 필수
- 보험료는 조건에 따라 다름 명시

**SEO 키워드 패턴**: `[보험] 추천`, `[보험] 비교`, `[보험] 청구 방법`

### 부동산/청약 정보 글 구조

```
1. 핵심 요약
2. 자격 조건/청약 점수
3. 신청 방법
4. 필요 서류
5. 일정/스케줄
6. FAQ
```

**SEO 키워드 패턴**: `[지역] 청약 일정`, `청약 점수 계산`, `전세 vs 월세`

## 재무설계/예산관리 글 작성 가이드

> **연계 계산기**: `/tools/budget-planner`

### 타겟 독자

| 페르소나 | 특징 | 검색 키워드 |
|---------|------|------------|
| 사회초년생 (24~28) | 첫 월급, 뭘 해야 할지 모름 | "첫 월급 관리", "사회초년생 저축" |
| 저축 고민 직장인 (28~35) | 돈이 새는 것 같음 | "저축률 높이는 법", "돈 안 모이는 이유" |
| 결혼/내집 준비 (30~38) | 목표 금액 있음 | "1억 모으기", "월급 관리 비율" |
| 적자 가계부 (전 연령) | 월급보다 지출 많음 | "적자 탈출", "지출 줄이기" |

### 글 톤 & 스타일

```
❌ 피해야 할 톤:
- "~에 대해 알아보겠습니다" (AI스러움)
- "반드시 ~해야 합니다" (설교조)
- "부자들은 이렇게 합니다" (현실과 동떨어짐)

✅ 사용할 톤:
- "월급 300인데 100도 못 모으겠다고요?" (공감)
- "솔직히 말하면, 카페 지출 줄이는 건 별로 효과 없어요" (현실적)
- "제가 실제로 해본 방법인데요" (경험 기반)
- "계산해보면 바로 답이 나와요" (계산기 연결)
```
